Task 1

The following sections will be computerized: recording services provided, managing member/provider record, generating reports, verifying members, producing running main accounting procedure, and tracking provider directory. The data will be stored in a repository that’s accessed remotely for each point of service system. Each point of service system will be a computer with a magstripe scanner attached. Additional hardware would be a database for storing the data of each member/provider and accounting information. The system will interact with the database, communications software, the EFT component, and the card reader. Users will interact with the system through a computer with the software installed.

Glossary

ChocAn:
Abbreviated form of "Chocoholics Anonymous".
ChocAn Data Center:
The physical location of the main ChocAn server.
ChocAn Member:
A person paying a monthly fee to Chocoholics Anonymous for services.
EFT:
An abbreviation for "electronic funds transfer".
EFT Data:
A file that contains the provider name, provider number, and the amount of funds to be transferred.
Main Accounting Procedure:
Weekly-run procedure (every Friday at midnight) that compiles reports for members and providers, and generates payment data.
Manager:
A ChocAn staff member who holds a management position.
Member:
A person who has joined an paid dues to ChocAn to benefit from the ChocAn providers' services.
Member Number:
A nine-digit number used to identify a member.
Member Status:
The current standing of a member within ChocAn.
Operator:
A ChocAn staff member who is in charge of updating member and provider information.
Provider:
A health care professional.
Provider Directory:
An alphabetically ordered list of service names with their corresponding service codes and fees.
Provider Number:
A nine-digit number used to identify a provider.
Record:
A file written to a disk that contains the following: current date and time, date service was provided, provider number, member number, service code, and comments.
Report:
A document sent via email to a member or a provider which contains a list of any services provided and the recipient's information.
Scheduler:
Mechanism that runs the main accounting procedure at pre-specified times.
Service Code:
A six-digit number used to identify a particular service.
Service Fee:
The amount that ChocAn owes the provider for services rendered.
Summary Report:
A printed report given to the accounts payable manager – this report contains every provider's number of consultations and fees to be paid for the week as well as the total number of providers who have provided services, the total number of consultations, and the overall total fees paid.
Suspended Member:
A member who has not paid dues for over a month.
Terminal:
A ChocAn computer terminal containing a card reader, keyboard input, and a single-line display.

Use Cases

Use case: Verify Membership

Context: The provider intends to verify the member attempting to use their service is in good standing with ChocAn

Actors: Provider

Main Success Scenario:

  1. 1.The system prompts the provider to enter the member's card number via the card reader or by manually entering the number.

  2. 2.The system validates the member number.

    1. 2.1.The system communicates with the ChocAn Data Center computer, which verifies the member number.

  3. 3.The system displays a success message.

Extensions:

  1. 2a.The member number is invalid

    1. 2a.1.The system displays the reason the number is not valid.

    2. 2a.2.The use case continues at step 1.

Use case: Record Service Provided

Context: The provider intends to enter the service they provided to the member into the ChocAn system.

Actors: Provider

Main Success Scenario:

  1. 1.The system prompts the provider to enter the member's card number via the card reader or by manually entering the number.

  2. 2.The system confirms the member number is valid.

    1. 2.1.The system communicates with the ChocAn Data Center computer, which verifies the member number.

  3. 3.The system prompts the provider to enter the date the service was provided in the format MM-DD-YYYY.

  4. 4.The provider enters the date.

  5. 5.The system prompts the provider to enter the service code corresponding to the service provided.

    1. 5.1.The system looks up the service name by code

  6. 6.The software product asks the provider if they wish to enter comments about the service.

  7. 7.The software product writes a record to disk which contains the following fields: Current date and time, Date service and provided, Provider number, Member number, Service code, and Comments.

  8. 8.The software displays the fee to be paid for that service.

    1. 8.1.The software looks up the fee to be paid.

  9. 9.The provider fills out their form as a personal record.

Extensions:

  1. 2a.The member number is invalid.

    1. 2a.1.The system displays the reason the number is not valid.

    2. 2a.2.The use case continues at step 1.

  2. 4a.The date is invalid.

    1. 4a.1.The system displays an error and the proper date format.

    2. 4a.2.The use case continues at step 3.

Use case: Run Main Accounting Procedure

Context: At midnight of every week, this procedure is run. This procedure reads through the week's services and prints reports.

  1. 1.The week's file of services are read.
